# 用户反馈：两题答案和解析核对结论

## 题一：S3/客户数据加密与访问隔离（id 696）

- **题干要点**：公司需为客户提供对 S3 中数据的安全访问；数据须静态加密；每个客户仅能从其 AWS 账户访问自己的数据；公司员工不得访问。
- **选项要点**：A/D 为 ACM + 客户端加密；B 为每客户 KMS + 服务端加密 + **S3 存储桶策略** 拒绝解密；C 为每客户 KMS + 服务端加密 + **KMS 密钥策略** 拒绝解密。

**数据核对结果：**

- 当前数据：`best_answer: "C"`，`vote_percentage: "73%"`。
- `explanation.analysis` 已含「本题以社区投票为准，答案为 C（社区 73%）」。
- `why_correct` 正确说明 C：KMS 密钥策略仅允许客户 IAM 角色解密，满足加密与隔离。
- `why_wrong` 正确说明 A/D 错在 ACM/客户端加密，B 错在 S3 存储桶策略无法像 KMS 密钥策略那样精确限制解密权限。

**结论：** 数据与解析一致，**无数据错误**。若用户看到「选项 C 错误」出现在「记住这点」中，属于**展示错误或旧版本**（当前 why_wrong 中未将 C 标为错误）。建议确认前端是否渲染了错误的 why_wrong 或混用了其他题目解析。

## 题二：Snowball Edge + HPC 低延迟访问（id 702）

- **题干要点**：200 TB 海洋调查数据在 Snowball Edge 存储优化设备上；AWS 上有 HPC 集群（石油天然气）；需为集群提供亚毫秒级延迟与高吞吐量访问；设备将发回 AWS。
- **选项要点**：A = S3 + Storage Gateway 文件网关；B = S3 + FSx for Lustre 与 S3 集成；C = S3 + EFS；D = 直接将数据导入 FSx for Lustre（不经 S3）。

**数据核对结果：**

- 当前数据：`best_answer: "D"`，`vote_percentage: "53%"`。
- `why_correct` 描述 D：直接将数据从 Snowball Edge 导入 FSx for Lustre，满足低延迟与高吞吐。
- `why_wrong` 中：B 被标为需先导入 S3 再经 FSx 访问，步骤更多；C 被标为 EFS 不适合 HPC。

**与用户截图的差异：**

- 用户截图中 App 将 **C 标为正确**、**B 标为错误**。
- 当前题库中答案为 **D**（社区 53%），与截图所示「C 正确」不一致。

**结论：** 存在**数据与用户所见不一致**的可能原因包括：（1）当前题库以社区 53% 为准记录为 D，用户端曾显示过旧数据或不同来源的答案 C；（2）或曾存在展示/缓存 bug 将正确答案显示为 C。建议以当前数据为准（答案为 D，解析已按 D 撰写），并在解析中保留「本题以社区投票为准，答案为 D（社区 53%），解析仅供参考」，若后续有权威来源或更高共识答案可再更新。
